About large test files

This directory contains test data that are too big for being committed on the GitHub repository. Only the directory structure is committed, with a CONTENT.txt file listing the expected content of each directory.

In order to enable the tests, the following file shall be provided:

<home directory>/<geotk directory>/Tests/coverage-sql.properties

Where <geotk directory> is:

.geotoolkit.org  on Linux systems
Library/Geotoolkit.org  on MacOS X systems
Application Data/Geotoolkit.org  on Windows systems

The file content shall be as below (the ... characters need to be replaced by information appropriate for the testing platform):

# --------------------------------------------------------
# Connection to the PostgreSQL server for testing purpose.
# Do NOT commit this file, since it contains a password.
# --------------------------------------------------------
server        = ...
database      = ...
schema        = coverages
user          = geoadmin
password      = ...
rootDirectory = ...

The rootDirectory value is a path to a directory containing the same sub-directories than this Tests directory, but we real data files in them. The rootDirectory value need also to be specified in the following files:

  • World/Coriolis/Aggregation.ncml

The test files in sub-directories are declared in the test coverages database by the coverages-data.sql script. For more information about Coverage-SQL package, see its readme page.
